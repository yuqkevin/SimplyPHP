Home:: SimplyPHP:: Dev Standard

Before we starting our sample practice, let's clarify or build some standard rules which we will follow in practice.

File Name: Lower Case String

We use lower case name for all files and folders except explicitly specified.

Class Name: Upper Camel Case String

  • All class names both in Model and Library are case-sensitive and in Upper Camel Case style. e.g MyModelClass
  • All library classes named with prefix "Lib",
    e.g. LibAcl, LibAclUser, LibSampleGbook, LibSampleBlogger, ..., etc.
  • All model classes named with prefix "Mo", but in URL, the prefix "Mo" is omitted.
    e.g. model MoAclUser's corresponding name in URL should be AclUser and vice versa.
  • The class name reflects hirarchy of file system under the bean structure.
    The first word (drop-off Lib prefix for library class) identify the bean folder (in lower case), and the followed words are sub-bean folder names (in lower case).
    For example: SampleBlogger (LibSampleBlogger) is model (library) for sub-bean blogger under bean sample.
    The file should be located at bean/sample/blogger/model.class.php (lib.class.php)
