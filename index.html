Welcome to SimplyPHP!

SimplyPHP is a simple, fast and scalable MVC PHP framework. Simplicity does not necessarily mean poor in power. On the contrary, Simplicity means straightforward, short learning curve for newbie, more flexibility for experienced developer.

Features

  • Scalable multi-tier architecture
  • MVC pattern
  • Component-oriented UI: Every request (URL) maps to a particular component
  • Business-oriented structure: All components are organized in business structures - W3S Bean
  • ORM/Active Record support: SQL is no longer mandatory for data access
  • Easy configuration: PHP-style configuration file
  • Simple and Straightforward: learning to write database-driven application in 15 minutes!

Key Concepts

  • MVC

    In SimplyPHP, Model is the core class dealing with client request. It is Component oriented, retrieving application data based on client request and invoked by Controller only.

    The View is presentation wrapper of data retrieved by Model.

    The Controller is minimized in SimplyPHP and not for cumstomizing. But it is the only interface to invoke and boot the application Model.

  • Component

    It is a function unit in UI. Page is a complex component which may contain multiple components. Every component is mapped to a pair of <Model, Handler>. From programmer's point of view, the Component is also an included handler within Model. So sometimes we also call handler as Method.

    In SimplyPHP, All requests are component request. The typical SimplyPHP request is something like http://yourappdomain.com/ModelName/HandlerName/...

  • Bean

    It is a structure contains files for particular application: Model(components), Library(business logic) and Database definition(tables).

  • Table Object

    In SimplyPHP, table (or view) which is defined in bean will be converted to object and hooked at Table Object Hook automatically when the Library is loaded. Record is associative array. Filter for searching is assorciative array. Record Set is linear array of Record. CURD are basic methods of Table Object, no SQL needed for typical query.
