Welcome to SimplyPHP!

SimplyPHP is a simple, fast and scalable MVC PHP framework. Simplicity does not necessarily mean poor in power. On the contrary, Simplicity means straightforward, short learning curve for newbie, more flexibility for experienced developer.

Features

  • Scalable multi-tier architecture
  • Simple UI Logic: Everything is component!
    • Both Page and Block in page are component in SimplyPHP.
    • A Component can contain other components.
    • Every request (URL) maps to a particular component, no matter it is a regualar request for page or an ajax request for widget.
  • MVC pattern but more powerful:
    • Controller is no longer a place where developer play around. It a tiny pice and shared by all applications.
    • View keeps the traditional definition and is presentation wrapper of data from Model
    • Model is data processor for client request. It is a Application-Oriented middleware.
    • Library is data service provider. It is Business-Oriented and based on database support.
  • Excellent application encapsulation: Application is a SimplyPHP Bean and has unique entrance in system application structure: The SimplyPHP Bean Tree.
  • ORM Access: Support Active Record pattern and Table Object to access database
  • Easy configuration: PHP-style configuration files
  • Cache: Component Level caching control and support multiple cache engines.
  • Multi-level Access Control: Model, Component and Action
  • The simplest RAD PHP framework: learning to write database-driven application in minutes!

Key Concepts

  • Component

    It is a function unit in UI. Page is a complex component which may contain multiple components. Every component is mapped to a pair of <Model, Handler>. From programmer's point of view, the Component is also an included handler within Model. So sometimes we also call handler as Method.

    In SimplyPHP, All requests are component request. The typical SimplyPHP request is something like http://yourappdomain.com/ModelName/HandlerName/...

  • SimplyPHP Bean

    A folder which contains Model, View and Library for the particular application. It implenents Application Encapsulation.

    Both Model and Library are object-oriented module, Model is request-oriented and can be inherited but can not be shared by other Models. Library is business-oriented, can be inherited and be shared by other Models and libraries.

  • Table Object

    In SimplyPHP, table (or view) which is defined in bean will be converted to object and hooked at Table Object Hook automatically when the Library is loaded. Record is associative array. Filter for searching is assorciative array. Record Set is linear array of Record. CURD are basic methods of Table Object, no SQL needed for typical query.
